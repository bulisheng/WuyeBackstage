- [使用文档](./使用文档.md)
- [框架文档](./框架文档.md)

## 1. 开发前准备

1. 使用微信开发者工具打开 `shengxing-property/project.config.json`。
2. 在 `miniprogram/setting/setting.js` 中配置你的 CloudBase 环境 ID。
3. 在 CloudBase 控制台创建并绑定云函数 `mcloud`。

## 2. 首次初始化

1. 打开小程序任意页面触发云函数调用。
2. 云函数会在首次运行时自动创建基础集合。
3. 初始数据会生成在首页、缴费、投诉、服务、商城、小区和房屋等集合中。

## 3. 认证流程

1. 用户在 `我的` 页提交业主认证。
2. 后台管理员在业主管理中审核。
3. 审核通过后，用户才可进入缴费、报修、投诉、物业服务、商城和客服工单等功能。

## 4. 后台管理员

1. 默认管理员账号仍沿用框架内置初始化逻辑。
2. 进入后台后先修改密码。
3. 再根据需要维护小区、房屋、账单、通知配置和业务数据。

## 5. 通知配置

1. 按 `docs/NOTICE_TEMPLATES.md` 规划通知模板。
2. 在后台填写钉钉机器人、短信和订阅消息配置。
3. 先从测试发送开始，确认 Webhook 和签名可用后再切到正式环境。

## 6. 当前实现范围

1. 已补齐缴费、投诉、物业服务、商城、客服、小区切换和房屋关系的基础闭环。
2. 已补齐业主认证后的权限开关。
3. 后续可继续在同一套目录结构下补后台管理页和更细的支付、催缴、通知、评价流程。
