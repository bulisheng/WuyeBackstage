# 02-后台系统规划.md

> 本文档定义后台布局、菜单、权限、小区切换器和模块开关的规划。适合在做后台页面前先看。

# 社区物业后台系统规划最终版

## 1. 后台定位

后台不是普通管理系统，而是物业 SaaS 的运营控制台。

核心目标：

- 管理多个小区
- 按小区隔离数据
- 按角色控制权限
- 按小区控制功能模块
- 统一处理工单
- 统一管理通知
- 支持运营数据分析

---

## 2. 后台全局布局

### 2.1 顶部区域

必须包含：

- 当前小区名称
- 小区切换器
- 当前登录管理员
- 当前角色
- 消息通知入口
- 退出登录

---

### 2.2 左侧菜单

菜单根据以下条件动态显示：

1. 当前管理员角色
2. 当前小区开启的模块
3. 当前管理员在该小区的权限

示例：

如果当前小区关闭商城，则后台不显示：

- 商城管理
- 商品管理
- 订单管理

如果当前管理员是财务，则只显示：

- 数据看板
- 缴费管理
- 支付记录
- 催缴管理

## 4. 小区切换器

### 4.1 功能

- 展示当前管理员可管理小区
- 支持切换小区
- 切换后刷新所有页面数据
- 切换后刷新左侧菜单
- 切换后刷新模块开关配置
- 无权限小区不可见

---

### 4.2 切换后影响范围

切换小区后，以下模块全部按当前小区过滤：

- 数据看板
- 用户与房屋
- 工单中心
- 缴费管理
- 商城管理
- 活动管理
- 投票管理
- 通知中心
- 客服中心
- 数据分析

---

### 4.3 安全要求

前端切换只是显示层。

后端必须根据管理员权限重新校验。

禁止通过修改 `current_community_id` 查看未授权小区。

## 6. 小区模块开关

### 6.1 页面位置

后台菜单：

```txt
系统配置 / 小区模块配置
```

也可以在：

```txt
小区管理 / 小区详情 / 模块配置
```

---

### 6.2 功能

- 查看当前小区所有模块
- 开启模块
- 关闭模块
- 设置模块展示名称
- 设置模块排序
- 设置模块配置参数
- 保存配置
- 恢复默认配置

---

### 6.3 模块列表

```txt
生活缴费
报事报修
投诉建议
物业服务
盛兴严选
在线客服
投票
社区活动
公告
Banner
```

---

### 6.4 配置影响

模块配置影响：

小程序端：

- 首页快捷入口
- 页面是否可访问
- TabBar 或功能入口显示
- 用户是否可以提交数据

后台端：

- 左侧菜单是否显示
- 管理页面是否可访问
- 是否允许新增数据

接口端：

- 是否允许调用新增、提交、支付、报名等接口

---

### 6.5 关闭模块规则

关闭模块后：

- 不删除历史数据
- 历史数据可查看
- 不允许新增数据
- 小程序隐藏入口
- 后台隐藏菜单
- 接口写操作返回错误

错误提示：

```txt
当前小区暂未开通该服务
```

### 16.2 小区授权

功能：

```txt

B 小区：维修人员

A 小区：财务
C 小区：小区管理员
```
