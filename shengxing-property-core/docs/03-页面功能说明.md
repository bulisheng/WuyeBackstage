# 03-页面功能说明.md

> 本文档定义小程序端和后台页面应如何动态展示、与小区/权限/模块开关如何联动。做页面前先看这篇。

> 当前进度：第一阶段页面已经完成，小程序首页和权限页都已经动态化；第二阶段正在补缴费页和工单页的闭环交互。

# 社区物业小程序与后台页面功能说明最终版

## 1. 全局说明

Codex 开发时必须理解：

本项目所有页面都受以下因素影响：

1. 当前小区
2. 当前用户身份
3. 当前小区模块开关
4. 当前用户权限
5. 当前数据状态

页面不能写死。

页面必须动态读取：

```txt
current_community_id
module_config
user_auth_status
user_houses
```

---

## 2. 小程序端全局规则

### 2.1 当前小区

用户进入小程序后：

- 识别用户已绑定房屋
- 根据房屋确定当前小区
- 如果绑定多个小区，允许切换当前小区
- 切换后首页和所有数据刷新

---

### 2.2 小程序模块显示规则

首页入口必须根据当前小区模块开关动态显示。

规则：

```txt
payment 开启            显示生活缴费
repair 开启             显示报事报修
complaint 开启          显示投诉建议
property_service 开启   显示物业服务
mall 开启               显示盛兴严选
customer_service 开启   显示在线客服
vote 开启               显示投票
activity 开启           显示社区活动
notice 开启             显示公告
banner 开启             显示 Banner
```

模块关闭时：

- 不显示入口
- 直接访问页面时提示：`当前小区暂未开通该服务`
- 不允许提交数据

---

### 2.3 用户认证规则

未认证用户：

- 可以查看首页
- 可以查看公告
- 可以进入认证页
- 不允许缴费
- 不允许报修
- 不允许投诉
- 不允许投票
- 不允许报名活动

已认证用户：

- 根据绑定房屋和模块开关使用服务

---

## 3. 小程序页面

## 3.1 首页

### 页面目标

作为用户进入小程序后的服务入口。

### 功能

- 展示当前小区名称
- 支持切换小区
- 展示 Banner
- 展示公告
- 展示快捷入口
- 展示待缴账单提醒
- 展示进行中工单提醒
- 展示活动提醒
- 展示认证引导

### 动态逻辑

未认证：

- 显示“请完成业主认证”
- 弱化缴费、报修等入口

有待缴账单：

- 首页展示待缴金额
- 高亮生活缴费入口

有进行中工单：

- 首页展示工单进度卡片

模块关闭：

- 不展示对应入口

# 5. 所有页面通用要求

每个页面必须实现：

- loading 状态
- empty 空状态
- error 错误状态
- success 成功反馈
- 权限不足提示
- 模块未开启提示
- 当前小区数据过滤
- 操作确认弹窗
- 状态标签
- 时间格式统一
- 金额格式统一

---

# 6. 错误提示规范

```txt
当前小区暂未开通该服务
您没有权限访问该小区
您没有权限操作该功能
数据不存在或已被删除
当前状态不允许该操作
请先完成业主认证
请先绑定房屋
```

# 7. Codex 开发重点

Codex 开发时必须优先实现：

1. `current_community_id` 全局机制
2. 小区切换器
3. 模块开关配置
4. 后台动态菜单
5. 小程序首页动态入口
6. API 权限中间件
7. 所有表 `community_id`
8. 统一工单 `tasks`
9. 统一通知 `notifications`
10. 状态流转日志
