# 盛兴物业使用文档

## 1. 项目启动

1. 用微信开发者工具打开 `shengxing-property/project.config.json`。
2. 配置 `miniprogram/setting/setting.js` 里的 `CLOUD_ID`。
3. 上传 `cloudfunctions/mcloud` 云函数并确保已绑定 CloudBase 环境。

## 2. 首次初始化

1. 打开小程序后，系统会自动补齐基础集合。
2. 登录后先完成业主认证。
3. 认证通过后才可以进入缴费、报修、投诉、物业服务、商城和客服功能。

## 3. 日常使用

### 3.1 业主端

- 在“我的”页提交认证资料。
- 在首页或服务页进入缴费、报修、投诉、商城和客服入口。
- 缴费页会先生成支付票据，再走支付回调完成状态落库。

### 3.2 管理端

- 在后台首页查看待办、统计和快捷入口。
- 账单、投诉、客服、订单都可以在详情页直接处理状态。
- 通知配置页可以编辑 Webhook、订阅消息模板和测试发送。

## 4. 后台操作建议

1. 先处理待审核业主。
2. 再处理待催缴账单、待回复投诉和待回复客服工单。
3. 商城订单优先看支付状态，再看发货和完成状态。
4. 通知配置先测试再正式启用。

## 5. 支付说明

1. 支付不再直接改状态，前端先拿到服务端生成的签名票据。
2. 前端提交 `payment/callback` 时必须携带 `timestamp`、`nonce`、`sign`。
3. 回调校验通过后才会更新账单或订单状态。

## 6. 导出说明

1. 账单列表页提供导出入口。
2. 导出后会生成云端文件链接。
3. 链接可复制到剪贴板后再下载或归档。

## 7. 常见问题

- 看不到业务入口：先确认业主认证状态是否通过。
- 支付后状态未变：检查回调参数签名是否完整。
- 通知未发送：检查后台通知配置是否启用，以及 Webhook/模板参数是否正确。
