# 云上部署与成本方案

这份文档面向“准备把系统放到云服务器、并支撑几万名用户”的场景，主要回答三个问题：

1. 登录过以后还要不要反复登录
2. 云服务器、数据库、域名、短信、接口调用大概要花多少钱
3. 哪一档方案适合当前这个物业系统

## 1. 登录态会不会反复登录

### 结论

- **正常情况下，登录过以后不需要每次都重新登录**
- 只要用户没有主动退出、没有清缓存、没有换设备，通常会一直保持登录态

### 现在代码里的做法

- 小程序登录成功后，会把这些信息写到本地存储：
  - `isLoggedIn`
  - `authToken`
  - `userInfo`
  - `communityInfo`
- 小程序启动时会从本地存储里恢复登录态，所以用户再次打开时通常可以直接进入
- 后端会把用户 token 和 openid 的对应关系持久化保存，不是只存在内存里
- 管理员登录有单独的会话过期时间，当前配置是 **720 分钟**

### 哪些情况会重新登录

- 用户主动退出
- 用户清理小程序缓存
- 更换手机或重新安装
- 后端保存的 token 记录被重置

### 对短信成本的影响

- 因为登录态会保留，所以**短信验证码主要花在首次登录、换机、找回密码、少量失效重登**
- 不是每次打开小程序都发短信

## 2. 费用怎么拆

下面的数字是**估算值**，实际以腾讯云、短信服务商和 DeepSeek 官方价格页为准。

### 先统一一下口径

- **用户规模**：按 `3 万 ~ 5 万注册用户` 来估
- **月活比例**：按 `10% ~ 20%` 的月活来估
- **短信场景**：主要算首次登录、换机、找回验证码，不按“每次打开都发短信”算
- **智能助手调用**：按 `5 万 ~ 10 万次 / 月` 的中等使用量来估

### 2.1 云服务器

建议分三档理解：

| 档位 | 推荐配置 | 适合场景 | 预算参考 |
|---|---|---|---|
| 试运行 | `2核4G` | 内测、演示、小规模试点 | 约 `130 ~ 250 元/月` |
| 正式上线 | `2核4G` 或 `4核8G` | 几万用户、日常使用 | 约 `260 ~ 450 元/月` |
| 更稳一些 | `4核8G` 起 | 用户量更大、并发更多 | 约 `450 元/月` 起 |

> 说明：云服务器价格会随地域、带宽、系统盘、购买时长变化，最终以云厂商价格计算器为准。

### 2.2 数据库

这个系统适合把主数据和业务数据都落库，数据库通常是最值得花钱的地方之一。

#### 低成本单节点

- 适合试运行
- 参考口径：`2GB + 20GB`
- 预算大约：`90 ~ 180 元/月`

#### 推荐的稳妥方案

- 适合正式上线
- 参考口径：`MongoDB 副本集 3 节点`
- 如果每个节点按 `4GB + 20GB` 估算：
  - 计算部分：`38.3333 × 4 × 3 ≈ 460 元/月`
  - 存储部分：`0.7000 × 20 × 3 = 42 元/月`
  - **合计约 `502 元/月`**

#### 更高一点的配置

- 如果每个节点按 `8GB + 50GB` 估算：
  - **合计约 `1,025 元/月`**

### 2.3 对象存储

对象存储主要放图片、附件、导出文件、头像等。

| 规模 | 预算参考 | 说明 |
|---|---|---|
| `10GB` | 约 `1 元/月` 左右 | 轻量使用 |
| `20GB` | 约 `2 元/月` 左右 | 常规使用 |
| `50GB` | 约 `5 元/月` 左右 | 图片和附件稍多 |

> 说明：这里只算存储本身，出网流量另算。

### 2.4 域名

- 如果你已经有域名，这部分可以忽略
- 如果新买域名，通常预算按 **`50 ~ 200 元/年`** 先预留
- 具体价格和后缀有关，`.com`、`.cn`、`.top` 等不同后缀差异很大

### 2.5 短信

短信一般只用在：

- 首次登录
- 找回验证码
- 换机重登
- 管理员或物业端需要二次验证

#### 估算公式

如果按常见短信单价 **`0.04 ~ 0.05 元/条`** 估算：

- `1,000` 条短信：约 `40 ~ 50 元`
- `10,000` 条短信：约 `400 ~ 500 元`
- `30,000` 条短信：约 `1,200 ~ 1,500 元`

#### 对几万用户的实际影响

如果系统有 **3 万名用户**，并且首批全部完成一次登录绑定：

- **一次性短信成本大约：`1,200 ~ 1,500 元`**

后续如果只是少量重新登录、换机、找回验证码：

- **每月短信成本通常会降到 `几十元到一两百元`**

### 2.6 智能助手接口调用

当前项目默认走 DeepSeek，调用方式是 OpenAI 兼容接口。

#### 估算公式

DeepSeek 按 token 计费时，可以用这个公式粗算：

- `月费用 ≈ 输入 token / 100万 × 输入单价 + 输出 token / 100万 × 输出单价`

当前文档按一个保守的估算口径：

- 输入：`2 元 / 100万 token`
- 输出：`8 元 / 100万 token`

#### 例子

如果一个月有 `5 万次` 智能助手调用，平均每次：

- `1000` 输入 token
- `200` 输出 token

那么月费用大约是：

- 输入：`50,000 × 1000 = 5000 万 token`，约 `100 元`
- 输出：`50,000 × 200 = 1000 万 token`，约 `80 元`
- **合计约 `180 元/月`**

如果一个月有 `10 万次` 调用：

- **合计大约 `360 元/月`**

如果达到 `30 万次` 调用：

- **合计大约 `1,080 元/月`**

> 说明：这里是保守估算，实际会受提示词长度、上下文长度、缓存命中率影响。

### 2.7 负载均衡

如果后面要做得更稳，可以加负载均衡。

- 早期单机可以先不加
- 如果要加，预算可先按 **`100 ~ 200 元/月`** 预留

## 3. 三套建议方案

### 方案 A：试运行

适合：

- 小区试点
- 内测
- 先把流程跑通

配置建议：

- 1 台 `2核4G` 云服务器
- 1 套单节点 MongoDB
- 20GB 左右对象存储
- 1 个域名
- 按需短信
- DeepSeek 智能助手

预算参考：

- **常态月费：约 `300 ~ 700 元/月`**
- **首月如果要一次性给 3 万用户做短信绑定，额外加 `1,200 ~ 1,500 元`**

### 方案 B：正式上线

适合：

- 几万名用户
- 正式小区使用
- 业务量已经稳定

配置建议：

- 1 台 `2核4G` 或 `4核8G` 云服务器
- MongoDB 副本集 3 节点
- 20GB 到 50GB 对象存储
- 1 个域名
- 按需短信
- DeepSeek 智能助手

预算参考：

- **常态月费：约 `900 ~ 1,800 元/月`**
- **首月如果要批量短信绑定，额外加 `1,200 ~ 1,500 元`**

### 方案 C：稳妥高可用

适合：

- 用户量更大
- 对稳定性要求更高
- 未来还要继续扩容

配置建议：

- 2 台云服务器 + 负载均衡
- MongoDB 副本集 3 节点更高配
- 更大的对象存储
- 按需短信
- DeepSeek 智能助手

预算参考：

- **常态月费：约 `1,600 ~ 3,500 元/月`**
- **首月批量短信成本另加**

## 4. 我建议你怎么选

如果目标是**几万用户正式上线**，建议优先选：

- **方案 B：正式上线**

理由：

- 成本和稳定性比较平衡
- 住户登录后通常不用反复登录，短信压力不会一直很高
- MongoDB 用副本集后，数据更稳
- DeepSeek 调用可以按实际使用量慢慢增加

如果你想先低成本试水，可以先走：

- **方案 A：试运行**

但如果后面准备正式推广，最好尽快切到方案 B。
