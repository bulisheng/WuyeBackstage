# 后台弹窗化编辑迁移记录

日期：2026-05-05

## 目标

把后台里分散在页面上的新增/编辑表单统一迁成弹窗，减少列表区与编辑区混排带来的视觉噪音，同时让保存成功/失败有明确反馈。

## 统一规范

- 弹窗标题统一用“新增 / 编辑 + 对象名称”，避免“新建”“保存修改”等多套说法混用。
- 弹窗底部统一用“取消 / 保存”，页面首屏只保留“新增”入口。
- 弹窗层级和间距统一由 `ModalDialog` 控制，避免每个页面自己调不同的 padding 和阴影。
- 保存成功与失败都在页面层给出明确提示，避免用户猜测结果。
- 本轮已完成按钮文案二次收口，去掉“保存账单 / 保存人员 / 保存配置”这类对象尾缀按钮，统一为“保存”。

## 运营使用说明

- 需要新增或编辑某类基础数据时，优先点击页面首屏的“新增”入口或列表里的“编辑”入口，再在弹窗里完成修改。
- 弹窗内的标题只负责告诉用户正在处理什么对象，底部按钮只负责确认与取消，不再重复对象名称。
- 如果页面同时存在“详情处理”“状态流转”“基础信息”这类动作，它们属于业务面板，不应再改回常驻编辑区。
- 保存后统一给出成功/失败提示，运营同事可以据此判断是否需要重试或联系后台。

## 现状

- 当前后台页面中，独立的新增/编辑表单已基本迁移到统一弹窗。
- 仍保留在页面中的 `OperationsPage`、`OwnersPage`、`ResidentsPage` 是业务处理或明细操作面板，不是独立新增/编辑页。
- `ResidentsPage` 继续保留导入和单条编辑动作的原有交互，因为它的编辑方式是单条属性修改，不是整块编辑表单。
- `RepairsPage` 保留工单基础信息和流转动作区，因为它属于报修详情处理面板，不是独立新增/编辑页。

## 备注

- 弹窗统一使用 `src/components/common/ModalDialog.vue`。
- 保存接口继续沿用原有 API，只在页面层负责打开/关闭弹窗和提示结果。
- 后续如果再新增后台表单页，默认先按本规范接入弹窗，不再新做页面内常驻编辑区。
