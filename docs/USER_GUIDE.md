# 使用者文档

这份文档给物业人员、运营人员和日常使用后台的人看。

## 你能用什么

### 微信小程序
- 手机号 + 验证码登录，后台按手机号自动匹配房屋
- 看首页公告、待缴账单、报修状态
- 提交报修
- 提交投诉/表扬
- 使用智能助手查物业费、查报修、生成报修草稿、生成投诉草稿
- 智能助手确认草稿后，可以直接跳到报修页或反馈页继续填写
- 看访客、装修、快递、代买菜等入口

### 首页怎么登录
- 业主只需要输入手机号和验证码，就可以登录。
- 后台会在 `住户账号 / 房屋档案` 里配置手机号与小区、楼栋、单元、房号的对应关系。
- 一个手机号可以对应多个房屋，一个房屋也可以对应多个用户。
- 如果同一个手机号匹配到多个房屋，登录时会先让用户选择房屋，再继续进入系统。
- 登录后，系统会自动把对应房屋信息带出来，不需要业主手动选小区、楼栋、单元、房号。
- 后台在配置时，`住户账号` 可维护多个 `关联房屋`，`房屋档案` 也可维护多个 `关联住户`。
- 如果手机号没有绑定任何房屋，页面会提示：`该手机号未绑定任何房屋，请联系物业处理`。

### Web 管理台
- 公告管理
- 账单管理
- 报修管理
- 投诉/表扬/反馈管理
- 智能助手的技能库、提示词和会话日志管理
- 访客管理
- 装修管理
- 住户账号管理
- 房屋档案管理
- 物业人员管理
- 小区管理
- 快递和蔬菜商品/订单管理
- 顶部会直接显示当前状态、当前项目列表、当前小区和项目数量
- 左侧控制台底部放了批量操作，方便直接选中当前页、清空选择和批量处理
- 顶部“当前项目列表”默认是折叠条，点一下才展开，页面更清爽

### 账单 / 物业费在哪配
- 物业费、物业管理费、水费、电费这类账单数据，都在 Web 管理台的 `账单管理` 里维护。
- 进入 `账单管理` 后，顶部会有一条更醒目的提示：`这里配置物业费`。
- 如果要按面积计费，先去 `房屋档案` 里维护每套房的 `面积`，再在 `账单管理` 里填 `每平米单价`，系统会自动按 `面积 × 单价` 算出金额。
- 先切到对应小区，再新增或编辑账单，填好 `标题、面积、每平米单价、金额、周期、到期日、状态、房号`。
- 小程序首页的待缴账单，会按当前住户绑定的房屋和当前小区来显示，不会跨小区串账单。
- 住户登录时只需要手机号和验证码；后台会根据已经配置好的 `住户账号 / 房屋档案` 自动关联到对应的小区、楼栋、单元和房号。
- 如果你要批量整理这些基础信息，建议先把 `手机号、小区、楼栋、单元、房号、面积、单价` 做成 CSV，再统一录入到住户 / 房屋 / 账单里。
- 如果你要改某个小区的物业费数据，优先看这三个地方：`当前小区`、`账单管理`、`房屋绑定`。

### 多小区怎么用
- 每个小区都有自己的“项目名称”“默认负责人”和“功能开关”。
- 小区管理支持新增 / 编辑都配置模块开关，保存后会按小区落到数据库，切换当前小区后前端会立即按当前小区的开关生效。
- 切换当前小区后，后台菜单和小程序入口会跟着变化。
- 小区里没开启的功能，前端不会显示入口。
- 当前小区切换后，住户、房屋、账单、报修、反馈等数据会按小区隔离，不会混到别的小区。
- 顶部“当前项目列表”会直接列出后台里真实存在的小区，方便核对现在有哪些项目。
- 如果顶部只显示折叠条，点一下“当前项目列表”就会展开小区按钮。

## 怎么用 Web 管理台

1. 打开浏览器访问 Web 管理台。
2. 输入接口地址和管理员密钥。
3. 登录后，从左侧菜单选择模块。
4. 在列表里搜索、筛选、排序。
5. 点某条记录查看详情、编辑或删除。
6. 支持批量操作的模块可以直接批量处理。
7. 顶部“当前项目列表”里可以直接切换当前小区。

## 怎么设负责人和通知对象

- `小区管理` 里设每个小区自己的默认负责人和负责人列表。
- `物业人员管理` 里给人员绑定飞书成员。
- `投诉规则` 里设负责人和通知对象。
- `智能配置` 里可以直接配置智能引擎，当前优先支持 `深度求索`，并且可以在后台改接口地址、接口密钥、模型名称和输出参数。
- `智能配置` 页现在按 `基础配置 / 引擎配置 / 通知配置 / 提示词配置` 四块展示，顶部有 `当前生效配置 vs 编辑中配置` 对比条，右侧有 `查看当前生效配置` 折叠面板，保存后会立即测试连接，页面会直接显示中文结果条，也可以复制测试结果。
- `智能配置` 页里的通知路由会直接显示 `绑定机器人 / 推送事项 / 负责人 / 备选负责人`，方便核对当前小区该推给谁。
- 只有绑定了 `feishuUserId` 的人员，才有机会被真正 `@` 到。
- `当前启用功能` 可以直接看这个项目开了哪些模块。
- `当前项目列表` 显示的是后台真实小区列表，不是写死的示例。

## 怎么看投诉流程

1. 住户提交投诉。
2. 后端写入投诉队列。
3. 系统做 AI 分析。
4. 再推到飞书群。
5. 如果飞书成员已绑定，就会尽量做真正的 `@`。
6. 现在飞书通知分成三路：
   - `客服通知机器人`：投诉 / 客诉
   - `维修通知机器人`：报修
   - `生活服务机器人`：快递代寄、蔬菜代买

### 哪些模块会发飞书
- **必须协同，才走飞书链接改状态**
  - 报修
  - 投诉 / 客服
  - 智能助手转人工
- **审批协同**
  - 装修
- **只提醒，不强制点链接**
  - 访客
  - 快递
  - 蔬菜订单
- **只在后台维护，不发飞书**
  - 账单
  - 住户
  - 房屋
  - 小区
  - 物业人员
  - 智能配置
  - FAQ

## 怎么用智能助手

1. 在首页点击 `智能助手`。
2. 直接输入问题，或者点快捷按钮。
3. 先试最常见的 4 类问题：
   - `查本月物业费`
   - `我有哪些待处理报修`
   - `帮我生成一条报修，水管漏水`
   - `帮我生成一条投诉，楼上太吵`
4. 如果 AI 生成了报修或投诉草稿，先确认再跳转到对应页面。
5. 在报修页或反馈页里补齐信息后提交。
6. 如果需要，随时点 `转人工`。
7. 在后台的 `会话日志` 里，可以看到这次智能助手对话的原始数据。
8. 在后台的 `提示词` 里，可以调智能助手的回答模板。
9. 在后台的 `AI 客服技能库` 里，可以按 `当前负责人`、`当前项目`、`标签` 过滤，也可以给每条技能单独指定负责人、同义问法、关键词和置顶顺序。
10. 技能库支持 CSV / JSON 导入，导入前会先预览，确认无误后再保存。
11. 技能库支持导出 CSV，也支持按场景下载模板，比如 `账单`、`报修`、`投诉`、`门禁`、`停车`。
12. 你还可以点 `批量生成同义问法`，系统会按问题和关键词自动补一批常见问法。

### 智能助手现在能不能测
- 可以测，当前小程序里已经有 `智能助手` 入口。
- 当前默认智能引擎已经切到 DeepSeek，并且已实测打通；如果你看到“尚未保存”或连接失败，先刷新后台再确认当前项目是否切对。
- 如果你要测“真正走智能引擎的回复”，需要先把后端、Web 管理台和本地智能引擎一起启动。
- 如果智能引擎暂时没起来，页面仍然可以打开，但部分回答会回退为本地规则或草稿模式。

### 智能助手测试步骤
1. 启动 MongoDB、后端 `server`、Web 管理台 `web-admin` 和本地智能引擎。
2. 微信里打开小程序，先确认已登录并绑定房屋。
3. 首页点击 `智能助手`。
4. 先发一句 `查本月物业费`，看是否直接返回账单摘要。
5. 再发一句 `帮我提报修，水管漏水`，看是否出现确认卡片。
6. 再发一句 `帮我生成一条投诉，楼上太吵`，看是否出现投诉草稿和跳转。
7. 再发一句 `转人工`，看是否进入人工接管流程。
8. 去后台 `会话日志` 看原始数据，确认智能引擎 / 回退逻辑是否正常。

## 怎么用 AI 客服技能库

1. 进入 Web 管理台，打开 `AI 客服技能库`。
2. 如果要新建一条，点 `新增问题`，填写问题、答案、标签、同义问法和关键词。
3. `同义问法` 里可以直接点 `自动补全模板`，先让系统补一批候选问法，再人工微调。
4. `批量生成同义问法` 可以把当前筛选范围里的多条技能一起补全。
5. 旁边的开关 `只补空同义问法` 开着时，只会补还没有同义问法的条目。
6. 点 `导入技能库` 后会先弹出预览窗口，确认无误再保存。
7. 导入时支持 CSV 和 JSON，CSV 更适合 Excel；导出也是 CSV。
8. 模板按钮里已经分了场景，你可以直接下载 `总模板`、`账单模板`、`报修模板`、`投诉模板`、`门禁模板`、`停车模板`。
9. 置顶的技能会排在更前面，适合先放最常问、最重要的内容。

## 常见注意事项

- `负责人` 只是规则负责人，不等于真正被通知的人。
- `通知对象` 才是飞书推送对象。
- 如果还没绑飞书标识，就只能发文字提醒。
- 多小区场景下，先切到正确的小区再配规则。
- 表扬和投诉已经统一进反馈管理，不要再去找单独的“表扬”老页面。
- 智能助手要测智能引擎时，优先确认智能引擎模式（`openclawMode`）、智能引擎地址（`openclawBaseUrl`）和本地服务都已启动。
- 技能库导入时，如果是模板文件，预览里会先显示前几条内容，确认后再点保存。
